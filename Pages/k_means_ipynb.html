class Kmeans:
    # intialise the class
    def __init__(self, k, seed=None, max_iterations=100):
        
        # define the internal variables
        self.k = k
        self.max_iterations = max_iterations
        self.centroids = None
        self.cluster_ids = None
      
        # if a seed is provided
        if seed is not None:
            # set the seed
            random.seed(seed) 
    
        
    # define the internal functions
    # -----------------------------
    
    def get_distance(self, point, centroid):
        
        # calculate Euclidean distance
        return statistics.fsum((x - y) ** 2 for x, y in zip(point, centroid)) ** 0.5

    def get_centroid(self, cluster):
        
        # calculate the mean of the points in the cluster        
        return [sum(points) / len(points) for points in zip(*cluster)]

    
    def k_means(self, data):
        
        
        # handle incorrect inputs
        # -----------------------
        
        # if data is not in list format
        if not isinstance(data, list):
            
                try:
                    # try to convert to list
                    data = data.values.tolist()
                    
                # if this fails raise error
                except AttributeError:
                    raise ValueError("Input data must be a list or a DataFrame")

        # if the data is empty 
        if len(data) == 0:
            # raise error
            raise ValueError("Input data cannot be empty.")    
            
        # if k is not an integer
        if not isinstance(self.k, int):
            try:
                 # try and convert it to integer
                self.k = int(self.k)
                
            # raise error is k cannot be converted
            except ValueError:  
                # catch the ValueError to show specific message
                raise ValueError("k must be an integer.")

                
        # start the clustering algorithm
        # ----------------------------------
            
        # choose k random centroids
        self.centroids = random.sample(data, self.k)

        # begin k means algorithm
        for _ in range(self.max_iterations):
            
            # create k empty clusters
            clusters = {i: [] for i in range(self.k)}

            # loop through all the data points
            for point in data:
                
                # calculate distances from point to all centroids
                distances = [self.get_distance(point, centroid) for centroid in self.centroids]
                
                # select the closest centroid
                closest_centroid = distances.index(min(distances))
                
                # assign the data point to that cluster
                clusters[closest_centroid].append(point)

            # recalculate the centroids
            new_centroids = []
            for cluster_points in clusters.values():
                new_centroids.append(self.get_centroid(cluster_points))

            # end the loop if the centroids do not change
            if new_centroids == self.centroids:
                break

            # rewrite the centroids with the new ones and restart loop
            self.centroids = new_centroids

            
        # map the data points to cluster IDs
        # ----------------------------------
        
        # flattern dictionary
        point_map = {str(point): cluster_id for cluster_id, points in clusters.items() for point in points}
        
        # assign cluster ids in order of data entries
        self.cluster_ids = [point_map[str(point)] for point in data]

        # return cluster id list and centroids
        return self.cluster_ids, self.centroids

Do you find any difference between the two measures?¶

The difference between the median and mean prices for each cluster highlights the skewness of the price distributions:

Cluster 0:
Median = 261
Mean = 422
The mean is substantially higher than the median, indicating a right-skewed distribution. This suggests that a few very high prices are pulling the mean up, while most prices are much lower.

Cluster 1:
Median = 287
Mean = 462 The same pattern is observed here, with the mean much higher than the median. This also indicates a right-skewed distribution.

Cluster 2:
Median = 193
Mean = 307
The mean remains higher than the median, showing the presence of outliers driving the distribution toward higher values.

Cluster 3:
Median = 185
Mean = 323
Similar to other clusters, the mean is much higher than the median, pointing to right skewness.

Interpretation¶

The consistent pattern across all clusters, where the mean is much higher than the median indicates right-skewed price distributions. This suggests that a small number of high outlier prices are impacting the average, while the median remains a more accurate measure of the central tendency.

Are there any differences across clusters?¶

Clusters 0 and 3 have the highest number of hotel room listings compared to the other clusters. This may suggest that these clusters correspond to the most popular destinations for holidaymakers, as hotel rooms are typically in demand in areas with high tourism activity.

Cluster 1, on the other hand, shows the lowest number of listings across all room types, suggesting that it is likely the smallest island or an area with lower overall accommodation demand. This could be due to lower tourist traffic, smaller population size, or fewer rental options.

Clusters 2 and 3 both have notable counts of shared room listings. This could potentially indicate areas with a higher concentration of hostels or backpacker accommodation options, catering to budget-conscious travellers.
